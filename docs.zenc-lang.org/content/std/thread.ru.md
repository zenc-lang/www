# std/thread

Модуль `std/thread` предоставляет высокоуровневый API для создания и управления потоками выполнения (threads), являясь безопасной оберткой над POSIX Threads (pthreads).

## Обзор

- **Создание потоков**: Запуск функций в отдельных потоках.
- **Управление жизненным циклом**: Ожидание завершения потоков (`join`) или их отсоединение (`detach`).
- **Идентификация**: Получение ID текущего потока.
- **Безопасность**: Интеграция с примитивами синхронизации из `std/sync`.

## Использование

```zc
import "std/thread.zc"
import "std/io.zc"

fn worker_func(arg: void*) -> void* {
    println "Привет из потока!";
    return null;
}

fn main() {
    let t = Thread::spawn(worker_func, null);
    
    // Ждем завершения
    t.join();
}
```

## Определения структур

### `Thread`
Представляет дескриптор работающего или завершенного потока.
```zc
struct Thread {
    handle: pthread_t;
    is_joined: bool;
}
```

## Методы

### Управление потоками

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **spawn** | `Thread::spawn(func: fn(void*) -> void*, arg: void*) -> Thread` | Создает и запускает новый поток. |
| **join** | `join(self) -> void*` | Блокирует текущий поток до завершения целевого. Возвращает результат работы. |
| **detach** | `detach(self)` | Отсоединяет поток, позволяя ему завершиться независимо. |
| **current_id**| `Thread::current_id() -> u64` | Возвращает числовой ID текущего потока. |
| **sleep** | `Thread::sleep(ms: int)` | Приостанавливает выполнение текущего потока на `ms` миллисекунд. |

## Управление памятью

Структура `Thread` реализует `impl Drop`. Если поток не был ни `join`, ни `detach` к моменту выхода структуры из области видимости, он будет автоматически отсоединен (`detach`), чтобы предотвратить утечку ресурсов.
