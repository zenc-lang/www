# std/thread

Модуль `std/thread` предоставляет высокоуровневые примитивы для создания и управления конкурентными потоками выполнения.

## Обзор

- **Нативные потоки**: Использует базовую поддержку потоков операционной системы (например, POSIX threads).
- **Поддержка замыканий**: `Thread::spawn` может принимать замыкания Zen-C, что позволяет легко обмениваться данными между потоками.
- **Явный жизненный цикл**: Потоки должны быть явно присоединены (`joined`) или отсоединены (`detached`) для обеспечения правильной очистки ресурсов.
- **Безопасность**: Об ошибках при создании или манипулировании потоками сообщается через `Result<bool>`.

## Использование

```zc
import "std/thread.zc"

fn worker(id: int) {
    println "Привет от воркера {id}";
}

fn main() {
    // Запуск с использованием замыкания
    let t = Thread::spawn(|| {
        worker(42);
    }).unwrap();
    
    // Явное ожидание завершения
    t.join();
}
```

## Определения структур

### `Thread`
Представляет дескриптор запущенного потока.
```zc
struct Thread {
    handle: void*;
}
```

## Методы

### Жизненный цикл `Thread`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **spawn** | `Thread::spawn(func: fn()) -> Result<Thread>` | Запускает новый поток, выполняющий предоставленное замыкание или функцию. |
| **join** | `join(self) -> Result<bool>` | Блокирует текущий поток до завершения запущенного потока. |
| **detach** | `detach(self) -> Result<bool>` | Отсоединяет поток, позволяя ему работать независимо. Ресурсы освобождаются автоматически при выходе. |
| **cancel** | `cancel(self) -> Result<bool>` | Отправляет запрос на отмену потока. |

### Вспомогательные функции

| Функция | Сигнатура | Описание |
| :--- | :--- | :--- |
| **sleep_ms** | `sleep_ms(ms: int)` | Приостанавливает выполнение текущего потока примерно на `ms` миллисекунд. |
