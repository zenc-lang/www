# std/sync

Модуль `std/sync` предоставляет полный набор примитивов синхронизации для управления конкурентным доступом к общим данным и координации выполнения потоков.

## Обзор

- **Стандартные примитивы**: Включает `Mutex`, `CondVar`, `RwLock`, `Once`, `Semaphore` и `Barrier`.
- **Интеграция с RAII**: Все примитивы реализуют трейт `Drop`, что гарантирует автоматическое освобождение системных ресурсов (таких как дескрипторы pthread).
- **Кроссплатформенность**: Безопасно абстрагирует специфические особенности платформ (например, реализация `Barrier` и `Semaphore` через мьютексы/условные переменные в macOS).
- **Эффективность**: Тонкие оболочки над оптимизированными системными библиотеками синхронизации.

## Использование

```zc
import "std/sync.zc"

fn main() {
    let m = Mutex::new();
    
    // Блокировка в области видимости (RAII)
    {
        m.lock();
        // Критическая секция
        m.unlock();
    } // m освобождается автоматически, если это был последний владелец
    
    // Однократная инициализация
    let once = Once::new();
    once.call(|| {
        println "Инициализировано!";
    });
}
```

## Определения структур

### `Mutex`
Мьютекс (взаимное исключение) для защиты общих данных.

### `CondVar`
Условная переменная для передачи сигналов между потоками на основе изменений состояния.

### `RwLock`
Блокировка чтения-записи, позволяющая иметь несколько одновременных читателей, но только одного писателя.

### `Once`
Гарантирует, что определенный кусок кода инициализации будет выполнен ровно один раз.

### `Semaphore`
Счетный семафор для управления доступом к пулу ресурсов.

### `Barrier`
Точка синхронизации, где несколько потоков должны ждать, пока не прибудет определенное их количество.

## Методы

### Методы `Mutex`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **new** | `Mutex::new() -> Mutex` | Создает новый мьютекс. |
| **lock** | `lock(self)` | Захватывает блокировку (блокирующий вызов). |
| **try_lock** | `try_lock(self) -> bool` | Пытается захватить блокировку без блокирования потока. |
| **unlock** | `unlock(self)` | Освобождает блокировку. |

### Методы `CondVar`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **new** | `CondVar::new() -> CondVar` | Создает новую условную переменную. |
| **wait** | `wait(self, mutex: Mutex*)` | Блокирует поток до получения сигнала, временно освобождая мьютекс. |
| **signal** | `signal(self)` | Пробуждает один поток, ожидающий на этом условии. |
| **broadcast**| `broadcast(self)` | Пробуждает все потоки, ожидающие на этом условии. |

### Методы `RwLock`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **new** | `RwLock::new() -> RwLock` | Создает новую блокировку чтения-записи. |
| **rdlock** | `rdlock(self)` | Захватывает общую блокировку на чтение. |
| **wrlock** | `wrlock(self)` | Захватывает эксклюзивную блокировку на запись. |
| **unlock** | `unlock(self)` | Освобождает любую удерживаемую блокировку. |

### Методы `Semaphore`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **new** | `Semaphore::new(value: int) -> Semaphore` | Создает новый семафор с начальным значением `value`. |
| **wait** | `wait(self)` | Уменьшает значение семафора (блокирует, если 0). |
| **post** | `post(self)` | Увеличивает значение семафора. |
| **value** | `value(self) -> int` | Возвращает текущее значение. |

### Методы `Barrier`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **new** | `Barrier::new(count: int) -> Barrier` | Создает новый барьер для потоков в количестве `count`. |
| **wait** | `wait(self) -> bool" | Ожидает у барьера. Возвращает `true` для назначенного "лидера". |

## Управление памятью

Все примитивы реализуют `impl Drop` и будут автоматически вызывать свой внутренний метод `free()` для освобождения системных ресурсов при выходе из области видимости.
