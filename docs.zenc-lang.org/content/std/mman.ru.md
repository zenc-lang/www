# std/sys/mman

Модуль `std/sys/mman` предоставляет Zen-C интерфейс для функций отображения памяти и защиты, оборачивая POSIX `sys/mman.h`.

## Обзор

- **Отображение памяти**: Отображение файлов или анонимной памяти в адресное пространство процесса.
- **Управление защитой**: Динамическое изменение прав доступа к областям памяти (чтение, запись, выполнение).
- **Анонимная память**: Выделение больших блоков памяти напрямую у ОС без использования файла.

## Использование

```zc
import "std/sys/mman.zc"
import "std/io.zc"

fn main() {
    let size: usize = 4096;
    let prot = Z_PROT_READ | Z_PROT_WRITE;
    let flags = Z_MAP_PRIVATE | Z_MAP_ANONYMOUS;
    
    let addr = Memory::mmap(size, prot, flags);
    if ((isize)addr == Z_MAP_FAILED) {
        println "Отображение не удалось";
        return;
    }
    
    // Использование памяти...
    
    Memory::munmap(addr, size);
}
```

## Методы

### Методы `Memory`

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **mmap** | `Memory::mmap(len: usize, prot: int, flags: int) -> void*` | Создает новое отображение в виртуальном адресном пространстве вызывающего процесса. |
| **munmap** | `Memory::munmap(addr: void*, len: usize) -> bool` | Удаляет отображения для указанного диапазона адресов. Возвращает `true` в случае успеха. |
| **mprotect** | `Memory::mprotect(addr: void*, len: usize, prot: int) -> bool` | Изменяет защиту доступа для страниц памяти вызывающего процесса. Возвращает `true` в случае успеха. |

## Константы

### Флаги защиты
- `Z_PROT_NONE`: Страница недоступна.
- `Z_PROT_READ`: Страница доступна для чтения.
- `Z_PROT_WRITE`: Страница доступна для записи.
- `Z_PROT_EXEC`: Страница доступна для выполнения.

### Флаги видимости
- `Z_MAP_SHARED`: Совместное использование этого отображения.
- `Z_MAP_PRIVATE`: Создание частного отображения с копированием при записи (copy-on-write).
- `Z_MAP_ANONYMOUS`: Отображение не связано ни с каким файлом.

### Значения ошибок
- `Z_MAP_FAILED`: Возвращается `mmap` в случае ошибки.
