# std/sys/signal

Модуль `std/sys/signal` предоставляет утилиты для работы с системными сигналами (такими как `SIGINT`, `SIGTERM` и др.), являясь оберткой над примитивами POSIX `signal.h`.

## Обзор

- **Обработчики сигналов**: Регистрируйте функции, которые будут вызваны при получении процессом определенного сигнала.
- **Стандартные сигналы**: Удобный доступ к константам стандартных сигналов.
- **Прерывание процессов**: Полезно для корректного завершения (graceful shutdown) приложений.

## Использование

```zc
import "std/sys/signal.zc"
import "std/io.zc"

fn on_interrupt(sig: int) {
    println "Получен сигнал {sig}. Завершение работы...";
    exit(0);
}

fn main() {
    // Регистрация обработчика для Ctrl+C
    Signal::handle(SIGINT, on_interrupt);
    
    // Бесконечный цикл
    while (true) {}
}
```

## Константы сигналов

| Константа | Описание |
| :--- | :--- |
| **SIGINT** | Прерывание с терминала (Ctrl+C). |
| **SIGTERM** | Сигнал завершения (корректная просьба закрыться). |
| **SIGKILL** | Немедленное завершение (невозможно перехватить). |
| **SIGSEGV** | Ошибка сегментации (доступ к чужой памяти). |
| **SIGABRT** | Сигнал аварийного прерывания (часто от `panic`). |

## Методы

### Управление сигналами

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **handle** | `Signal::handle(sig: int, handler: fn(int))` | Устанавливает функцию-обработчик для сигнала `sig`. |
| **raise** | `Signal::raise(sig: int) -> int` | Посылает сигнал `sig` самому себе (текущему процессу). |
| **ignore** | `Signal::ignore(sig: int)` | Настраивает процесс на полное игнорирование сигнала `sig`. |
| **reset** | `Signal::reset(sig: int)` | Восстанавливает стандартное системное поведение для сигнала `sig`. |
