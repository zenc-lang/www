# std/env

Модуль `std/env` обеспечивает кроссплатформенный доступ к переменным окружения процесса.

## Обзор

- **Доступ по ключу**: Простой API для получения, установки и удаления переменных окружения.
- **Заимствованные или собственные**: Выбор между `get` (возвращает заимствованную C-строку) и `get_dup` (возвращает собственную `String`, выделенную в куче).
- **Кроссплатформенность**: Безопасно абстрагирует системные вызовы для манипулирования окружением.

## Использование

```zc
import "std/env.zc"

fn main() {
    // Установка переменной окружения
    Env::set("MY_APP_MODE", "development");

    // Получение (заимствованное)
    match Env::get("MY_APP_MODE") {
        Some(val) => println "Mode: {val}",
        None => println "Mode not set"
    }

    // Получение (собственная строка String для RAII)
    match Env::get_dup("HOME") {
        Some(home) => {
            println "Home: {home}";
            // home освобождается автоматически
        }
        None => println "HOME not found"
    }
}
```

## Определение перечисления

```zc
enum EnvRes {
    OK,
    ERR,
}
```

## Методы

### Доступ и запросы

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **get** | `Env::get(name: char*) -> Option<char*>` | Возвращает заимствованный указатель на переменную окружения. Не освобождайте его вручную. |
| **get_dup** | `Env::get_dup(name: char*) -> Option<String>` | Возвращает переменную окружения как новый объект `String`. |

### Модификация

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **set** | `Env::set(name: char*, value: char*) -> EnvRes` | Устанавливает или обновляет переменную окружения. |
| **unset** | `Env::unset(name: char*) -> EnvRes` | Удаляет переменную окружения из текущего процесса. |
