# std/result

`Result<T>` — это стандартный тип для обработки ошибок в Zen-C. Он представляет либо успех (`Ok`), содержащий значение типа `T`, либо неудачу (`Err`), содержащую строку с сообщением об ошибке.

## Обзор

- **Безопасность**: Требует явной обработки путей успеха и ошибки.
- **Информативность**: Варианты `Err` несут текстовое описание причины ошибки.
- **Обобщенность**: Поддерживает любой тип значения `T` для успешного результата.
- **Интеграция**: Отлично работает с макросами и паттернами Zen-C для лаконичной передачи (propagation) ошибок.

## Использование

```zc
import "std/result.zc"

fn divide(a: int, b: int) -> Result<int> {
    if (b == 0) {
        return Result<int>::Err("Деление на ноль");
    }
    return Result<int>::Ok(a / b);
}

fn main() {
    match divide(10, 0) {
        Ok(val) => println "Результат: {val}",
        Err(e)  => println "Ошибка: {e}"
    }
}
```

## Определение структуры

```zc
struct Result<T> {
    is_ok: bool;
    val: T;
    err: char*;
}
```

## Методы

### Создание

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **Ok** | `Result<T>::Ok(v: T) -> Result<T>` | Создает успешный результат, содержащий `v`. |
| **Err** | `Result<T>::Err(e: char*) -> Result<T>` | Создает результат с ошибкой и сообщением `e`. |

### Проверка

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **is_ok** | `is_ok(self) -> bool` | Возвращает `true`, если результат `Ok`. |
| **is_err** | `is_err(self) -> bool` | Возвращает `true`, если результат `Err`. |

### Извлечение

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **unwrap** | `unwrap(self) -> T` | Возвращает значение успеха. Паникует с текстом ошибки, если это `Err`. |
| **unwrap_ref** | `unwrap_ref(self) -> T*` | Возвращает указатель на значение успеха. Паникует, если это `Err`. |
| **expect** | `expect(self, msg: char*) -> T` | Возвращает значение или паникует с `msg` и текстом ошибки, если это `Err`. |

| :--- | :--- | :--- |
| **forget** | `forget(self)` | Очищает значение успеха без вызова деструкторов или освобождения памяти. |
