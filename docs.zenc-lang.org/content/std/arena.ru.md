# std/arena

Модуль `std/arena` предоставляет аллокатор памяти на основе регионов (арена), который позволяет осуществлять быструю аллокацию и эффективное освобождение всей памяти за один раз.

## Обзор

- **Быстрая аллокация**: Простое увеличение указателя внутри предварительно выделенных блоков памяти.
- **Эффективное освобождение**: Вся память, выделенная в арене, может быть освобождена одной операцией (`free`).
- **Безопасность**: Помогает избежать утечек памяти и фрагментации путем управления жизненным циклом связанных объектов целиком.

## Использование

```zc
import "std/arena.zc"

fn main() {
    // Создаем новую арену
    let a = Arena::new();

    // Выделяем память из арены
    let p1 = a.alloc(100);
    let p2 = a.alloc_type(int);

    // В конце области видимости вся память арены автоматически освобождается через trait Drop
}
```

## Определение структуры

```zc
struct Arena {
    blocks: Vec<void*>;
    current_block: void*;
    block_size: usize;
    offset: usize;
}
```

## Методы

### Создание

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **new** | `Arena::new() -> Arena` | Создает новую арену с размером блока по умолчанию (4096 байт). |
| **with_capacity** | `Arena::with_capacity(size: usize) -> Arena` | Создает новую арену с указанным размером блока. |

### Аллокация

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **alloc** | `alloc(self, size: usize) -> void*` | Выделяет `size` байт из арены. |
| **alloc_type** | `alloc_type(self, type T) -> T*` | Макрос для удобного выделения памяти под тип `T`. |
| **alloc_array** | `alloc_array(self, type T, count: usize) -> T*` | Выделяет память под массив из `count` элементов типа `T`. |

### Управление

| Метод | Сигнатура | Описание |
| :--- | :--- | :--- |
| **clear** | `clear(self)` | Освобождает всю память, выделенную в арене, не уничтожая сам объект арены. |
| **free** | `free(self)` | Вручную освобождает всю память и внутренние блоки. |
| **Trait** | `impl Drop for Arena` | Автоматически вызывает `free()` при выходе из области видимости. |

## Детали реализации

- Арена использует связанный список или вектор блоков памяти.
- Аллокации выравниваются в соответствии с требованиями архитектуры.
- Когда текущий блок заполняется, выделяется новый блок.
