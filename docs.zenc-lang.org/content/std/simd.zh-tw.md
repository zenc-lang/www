# std/simd

Zen-C 提供原生的 SIMD（單指令多數據）向量類型，可直接編譯為目標後端支援的原生硬體優化向量指令（如 SSE、AVX、NEON 等）。

## 概覽

- **原生效能**：利用 LLVM/GCC 向量擴展實現極高效率。
- **隱式可移植性**：如 `f32x4` 等類型會根據架構映射至最佳的 128 位元硬體指令。
- **逐元素運算**：標準運算子（`+`、`-`、`*`、`/`）同時作用於所有通道（lanes）。
- **廣播（Broadcasting）**：使用單個值初始化時，該值會被廣播至所有通道。

## 使用方法

```zc
import "std/simd.zc"

fn main() {
    // 初始化（顯式指定各通道值）
    let a = f32x4 { 1.0, 2.0, 3.0, 4.0 };
    
    // 廣播（將單個值應用於所有通道）
    let b = f32x4 { v: 2.0 };
    
    // 逐元素加法
    let c = a + b;   // 結果：{ 3.0, 4.0, 5.0, 6.0 }
    
    // 通道存取
    let first = c[0];
}
```

## 向量類型

標準庫定義了多種 128 位元和 256 位元向量類型。您也可以使用 `@vector(N)` 特性定義自定義類型。

### 128 位元向量 (SSE / NEON)

| 類型 | 元素類型 | 通道數 | 位元組大小 |
| :--- | :--- | :--- | :--- |
| `f32x4` | `f32` | 4 | 16 |
| `f64x2` | `f64` | 2 | 16 |
| `i32x4` | `i32` | 4 | 16 |
| `u32x4` | `u32` | 4 | 16 |
| `i64x2` | `i64` | 2 | 16 |
| `u64x2` | `u64` | 2 | 16 |
| `i16x8` | `i16` | 8 | 16 |
| `u16x8` | `u16` | 8 | 16 |
| `i8x16` | `i8` | 16 | 16 |
| `u8x16` | `u8` | 16 | 16 |

### 256 位元向量 (AVX / AVX2)

| 類型 | 元素類型 | 通道數 | 位元組大小 |
| :--- | :--- | :--- | :--- |
| `f32x8` | `f32` | 8 | 32 |
| `f64x4` | `f64` | 4 | 32 |
| `i32x8` | `i32` | 8 | 32 |
| `u32x8` | `u32` | 8 | 32 |
| `i64x4` | `i64` | 4 | 32 |
| `u64x4` | `u64` | 4 | 32 |
| `i16x16` | `i16` | 16 | 32 |
| `u16x16` | `u16` | 16 | 32 |
| `i8x32` | `i8` | 32 | 32 |
| `u8x32` | `u8` | 32 | 32 |

## 操作

| 類別 | 運算子 | 說明 |
| :--- | :--- | :--- |
| **算術** | `+`, `-`, `*`, `/` | 標準的逐元素加、減、乘、除。 |
| **位元** | `&`, `|`, `^`, `~` | 跨所有通道的位元與 (AND)、或 (OR)、互斥或 (XOR) 及反向 (NOT)。 |
| **索引** | `[i]` | 通過索引存取或修改單個通道。 |
| **比較** | `==`, `!=`, `<`, `>` | 返回布林遮罩向量（結果視後端而異）。 |
