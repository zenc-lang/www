# 15. 诊断系统

Zen C 拥有一个分类诊断系统，可以对编译器警告进行细粒度控制。这使您能够在与外部 C 代码交互时减少摩擦，同时保持高水平的代码质量标准。

## 诊断类别

警告按逻辑类别分组。可以使用编译器标志全局启用或禁用每个类别。

| 类别 | 描述 | 默认值 |
| :--- | :--- | :--- |
| **`INTEROP`** | 与 C 头文件导入和未定义的外部函数相关的警告。 | **关闭** |
| **`PEDANTIC`** | 对潜在问题或代码质量的额外严格检查。 | **关闭** |
| **`UNUSED`** | 对已定义但未使用的变量、参数或函数的警告。 | **开启** |
| **`SAFETY`** | 关键安全警告，如空指针访问或除零。 | **开启** |
| **`LOGIC`** | 与逻辑相关的警告，如不可达代码或常量比较。 | **开启** |
| **`CONVERSION`** | 隐式或收窄类型转换的警告。 | **开启** |
| **`STYLE`** | 编码风格警告，如变量遮蔽。 | **开启** |

## 编译器标志

您可以使用 `-W`（启用）和 `-Wno-`（禁用）标志，后跟类别名称或特定诊断 ID 来控制诊断。

### 类别标志

- `-Winterop`: 启用所有与互操作相关的警告。
- `-Wno-unused`: 特别静默未使用变量/参数的警告。
- `-Wsafety`: 确保所有安全检查都处于活动状态。
- `-Wall`: 启用所有主要的诊断类别。
- `-Wextra`: 启用更严格的诊断（相当于 `-Wpedantic`）。

### 使用示例

```bash
# 在启用 C 互操作警告的情况下编译
zc app.zc -Winterop

# 在启用所有警告但禁用未使用警告的情况下编译
zc app.zc -Wall -Wno-unused
```

## C 互操作摩擦

默认情况下，Zen C 会抑制可能在 C 标准库中找到的函数的“未定义函数”警告（`INTEROP` 类别默认为 **关闭**）。

如果您希望编译器严格报告每个未定义的函数（例如，为了捕捉拼写错误），请启用互操作类别：

```bash
zc main.zc -Winterop
```

启用后，编译器将为常见的 C 函数提供有用的提示：
```text
warning: Undefined function 'abs'
  --> main.zc:5:13
   |
5  |     let x = abs(-5);
   |             ^ here
   |
   = note: If this is a C function, it might need to be whitelisted in 'zenc.json'
```

## 白名单

如果您经常使用特定的 C 库并希望在保持 `-Winterop` 启用的同时不受特定函数的干扰，可以将它们添加到 `zenc.json` 配置文件中的 `c_function_whitelist` 中。
