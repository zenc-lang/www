# 15. Система диагностики

Zen C оснащен категоризированной системой диагностики, которая обеспечивает детализированный контроль над предупреждениями компилятора. Это позволяет поддерживать высокие стандарты качества кода, снижая при этом трение при взаимодействии с внешним C-кодом.

## Категории диагностики

Предупреждения сгруппированы в логические категории. Каждую категорию можно включить или выключить глобально с помощью флагов компилятора.

| Категория | Описание | По умолчанию |
| :--- | :--- | :--- |
| **`INTEROP`** | Предупреждения, связанные с импортом C-заголовков и неопределенными внешними функциями. | **ВЫКЛ** |
| **`PEDANTIC`** | Особо строгие проверки потенциальных проблем или качества кода. | **ВЫКЛ** |
| **`UNUSED`** | Предупреждения для определенных, но неиспользуемых переменных, параметров или функций. | **ВКЛ** |
| **`SAFETY`** | Критические предупреждения по безопасности, такие как доступ по нулевому указателю или деление на ноль. | **ВКЛ** |
| **`LOGIC`** | Предупреждения, связанные с логикой, такие как недостижимый код или сравнения констант. | **ВКЛ** |
| **`CONVERSION`** | Предупреждения для неявных или сужающих преобразований типов. | **ВКЛ** |
| **`STYLE`** | Предупреждения о стиле программирования, такие как скрытие переменных (shadowing). | **ВКЛ** |

## Флаги компилятора

Вы можете управлять диагностикой с помощью флагов `-W` (включить) и `-Wno-` (выключить), за которыми следует имя категории или конкретный идентификатор диагностики.

### Флаги категорий

- `-Winterop`: Включить все предупреждения, связанные с совместимостью.
- `-Wno-unused`: Специально подавлять предупреждения о неиспользуемых переменных/параметрах.
- `-Wsafety`: Убедиться, что все проверки безопасности активны.
- `-Wall`: Включить все основные категории диагностики.
- `-Wextra`: Включить еще более строгую диагностику (эквивалентно `-Wpedantic`).

### Пример использования

```bash
# Компиляция с включенными предупреждениями о совместимости с C
zc app.zc -Winterop

# Компиляция со всеми включенными предупреждениями, но без предупреждений о неиспользуемом коде
zc app.zc -Wall -Wno-unused
```

## Трение при взаимодействии с C

По умолчанию Zen C подавляет предупреждения "Неопределенная функция" для функций, которые, скорее всего, будут найдены в стандартных библиотеках C (категория `INTEROP` по умолчанию **ВЫКЛ**).

Если вы хотите, чтобы компилятор строго сообщал о каждой неопределенной функции (например, чтобы поймать опечатки), включите категорию совместимости:

```bash
zc main.zc -Winterop
```

При включении этой категории компилятор будет давать полезные подсказки для распространенных функций C:
```text
warning: Undefined function 'abs'
  --> main.zc:5:13
   |
5  |     let x = abs(-5);
   |             ^ here
   |
   = note: If this is a C function, it might need to be whitelisted in 'zenc.json'
```

## Белый список

Если вы часто используете определенную библиотеку C и хотите оставить `-Winterop` включенным, не беспокоясь о конкретных функциях, вы можете добавить их в `c_function_whitelist` в вашем конфигурационном файле `zenc.json`.
