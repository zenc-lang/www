# 19. 診斷系統

Zen C 擁有一個分類診斷系統，可以對編譯器警告进行細粒度控制。這使您能夠在與外部 C 代碼交互時減少摩擦，同時保持高水平的代碼質量標準。

## 診斷類別

警告按邏輯類別分組。可以使用編譯器標誌全局啟用或禁用每個類別。

| 類別 | 描述 | 默認值 |
| :--- | :--- | :--- |
| **`INTEROP`** | 與 C 頭文件導入和未定義的外部函數相關的警告。 | **關閉** |
| **`PEDANTIC`** | 對潛在問題或代碼質量的額外嚴格檢查。 | **關閉** |
| **`UNUSED`** | 對已定義但未使用的變量、參數或函數的警告。 | **開啟** |
| **`SAFETY`** | 關鍵安全警告，如空指針訪問或除零。 | **開啟** |
| **`LOGIC`** | 與邏輯相關的警告，如不可達代碼或常量比較。 | **開啟** |
| **`CONVERSION`** | 隱式或收窄類型轉換的警告。 | **開啟** |
| **`STYLE`** | 編碼風格警告，如變量遮蔽。 | **開啟** |

## 編譯器標誌

您可以使用 `-W`（啟用）和 `-Wno-`（禁用）標誌，後跟類別名稱或特定診斷 ID 來控制診斷。

### 類別標誌

- `-Winterop`: 啟用所有與互操作相關的警告。
- `-Wno-unused`: 特別靜默未使用變量/參數的警告。
- `-Wsafety`: 確保所有安全檢查都處於活動狀態。
- `-Wall`: 啟用所有主要的診斷類別。
- `-Wextra`: 啟用更嚴格的診斷（相當於 `-Wpedantic`）。

### 使用示例

```bash
# 在啟用 C 互操作警告的情況下編譯
zc app.zc -Winterop

# 在啟用所有警告但禁用未使用警告的情況下編譯
zc app.zc -Wall -Wno-unused
```

## C 互操作摩擦

默認情況下，Zen C 會抑制可能在 C 標準庫中找到的函數的“未定義函數”警告（`INTEROP` 類別默認為 **關閉**）。

如果您希望編譯器嚴格報告每個未定義的函數（例如，為了捕捉拼寫錯誤），請啟用互操作類別：

```bash
zc main.zc -Winterop
```

啟用後，編譯器將為常見的 C 函數提供有用的提示：
```text
warning: Undefined function 'abs'
  --> main.zc:5:13
   |
5  |     let x = abs(-5);
   |             ^ here
   |
   = note: If this is a C function, it might need to be whitelisted in 'zenc.json'
```

## 白名單

如果您經常使用特定的 C 庫並希望在保持 `-Winterop` 啟用的同時不受特定函數的干擾，可以將它們添加到 `zenc.json` 配置文件中的 `c_function_whitelist` 中。
