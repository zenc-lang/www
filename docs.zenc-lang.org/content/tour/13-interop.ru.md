# 17. Совместимость с C


Zen C предлагает два способа взаимодействия с C-кодом: **Доверенный импорт** (удобно) и **Явный FFI** (безопасно/точно).

#### Способ 1: Доверенный импорт (удобно)

Вы можете импортировать C-заголовок напрямую, используя ключевое слово `import` с расширением `.h`. Это рассматривает заголовок как модуль и предполагает, что все символы, к которым осуществляется доступ через него, существуют.

```zc
//> link: -lm
import "math.h" as c_math;

fn main() {
    // Компилятор доверяет корректности; генерирует 'cos(...)' напрямую
    let x = c_math::cos(3.14159);
}
```

> **Плюсы**: Ноль шаблонного кода. Доступ ко всему в заголовке сразу.
> **Минусы**: Отсутствие типизации со стороны Zen C (ошибки отлавливаются компилятором C позже).

#### Способ 2: Явный FFI (безопасно)

Для строгой проверки типов или когда вы не хотите включать текст заголовка, используйте `extern fn`.

```zc
include <stdio.h> // Генерирует #include <stdio.h> в созданном C-коде

// Определение строгой сигнатуры
extern fn printf(fmt: char*, ...) -> c_int;

fn main() {
    printf("Hello FFI: %d\n", 42); // Типизация проверяется Zen C
}
```

> **Плюсы**: Zen C гарантирует соответствие типов.
> **Минусы**: Требуется ручное объявление функций.

#### `import` vs `include`

- **`import "file.h"`**: Регистрирует заголовок как именованный модуль. Обеспечивает неявный доступ к символам (например, `file::function()`).
- **`include <file.h>`**: Просто вставляет `#include <file.h>` в сгенерированный C-код. Не вводит никаких символов в компилятор Zen C; вы должны использовать `extern fn` для доступа к ним.

#### Диагностика и трение

По умолчанию Zen C подавляет предупреждения "Неопределенная функция" для стандартных функций библиотеки C, чтобы обеспечить беспрепятственное взаимодействие. Вы можете управлять этим поведением с помощью флагов `-Winterop` и `-Wno-interop`.

[Подробнее о системе диагностики](15-diagnostics.ru.md)
