# 13. Взаимодействие с C


Zen C предлагает два способа взаимодействия с кодом на C: **Trusted Imports** (удобно) и **Explicit FFI** (безопасно/точно).

#### Способ 1: Trusted Imports (удобно)
Вы можете импортировать заголовочный файл C напрямую, используя ключевое слово `import` с расширением `.h`. Это рассматривает заголовок как модуль и предполагает, что все доступные символы существуют.

```zc
//> link: -lm
import "math.h" as c_math;

fn main() {
    // Компилятор доверяет корректности; напрямую выводит 'cos(...)'
    let x = c_math::cos(3.14159);
}
```

> **Плюсы**: Нет лишнего кода. Мгновенный доступ ко всему в заголовке.
> **Минусы**: Нет проверки типов со стороны Zen C (ошибки будут обнаружены позже C-компилятором).

#### Способ 2: Явное FFI (безопасно)
Для строгого контроля типов или когда вы не хотите включать текст заголовка, используйте `extern fn`.

```zc
include <stdio.h> // Выводит #include <stdio.h> в сгенерированный C
// Определите строгую сигнатуру
extern fn printf(fmt: char*, ...) -> c_int;

fn main() {
    printf("Привет, FFI: %d\n", 42); // Типы проверяются Zen C
}
```

> **Плюсы**: Zen C гарантирует соответствие типов.
> **Минусы**: Требует ручного объявления функций.

#### import против include
- **import "file.h"**: Регистрирует заголовок как именованный модуль; обеспечивает неявный доступ к символам (например, file::function()).
- **include <file.h>**: Просто выводит #include <file.h> в сгенерированный код C; не вводит никаких символов в компилятор Zen C (вы должны использовать extern fn).
